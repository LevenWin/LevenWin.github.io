透视HTTP-历史

因为计算机需要互相通信，所以HTTP便诞生了。

HTTP协议定义了计算机如何通信的基础规则。早期是在美国国防部的ARPA建立的局域网内使用。初期研究员发明了TCP/IP协议。因为具有良好的分层结构和可拓展性。迅速得到了广泛的使用，促使更多的计算机介入了互联网。

创世纪

1989年任职欧洲核子研究中心的Tim Berners-Lee发表了一篇论文，提出了在互联网上构建超链接文档系统的构想。提出了下面三个概念

  1. URI：统一资源标识符，作为互联网身份的唯一标识
  2. HTML：超文本标记语言，描述超文本文档
  3. HTTP：即传输超文本传输协议，用来传输超文本
    基于上面三个概念，能够让世界各地的人们互相共享信息。Tim将这个称为万维网

HTTP/0.9

20世纪90年，互联网世界非常简陋。计算机的性能和储存容量都很差劲。大多数的资源都是文本资源。此时HTTP只允许“GET”操作从服务器获取到文本。并且响应请求后立即关闭连接。此时是0.9版

HTTP/1.0

199X年这个时期出现了Apache服务器，简化了服务器搭建工作。同时互联网上的内容更加丰富了，出现了JPEG图片，MP3音乐。新媒体内容的出现极大的吸引了更多的初代网民。基于之前的HTTP实践，HTTP/1.0版本在1996年发布，多方面增强了9.0，例如：

  1. 增加了HEAD，POST
  2. 增加了响应状态码
  3. 引入协议版本号
  4. 引入了HTTP Header。
  5. 传输的数据不再局限于文本

但是HTTP/1.0不是一个必须执行的标准，没有约束力。直到HTTP/1.1的出现

HTTP/1.1

1999年，HTTP/1.1发布了RFC文档。这是个对1.0的小幅修正，最大的区别在于1.1是一个正式的标准，而不是参考文档。所以要接入互联网的设备必须遵守这个协议。主要个改例如：

  1. 增加了PUT，DELETE等新方法
  2. 增加了缓存机制
  3. 明确了连接管理和允许持久连接
  4. 允许相应数据分块chunked，利于传输大文件
  5. 强制要求HOST头，让互联网主机托管称为可能

HTTP/1.1的出现极大的促进了互联网的发展，中国各大门户网站也几乎在这个时期成立了。随着资源越来越丰富和冗杂。HTTP/1.1连接慢的问题慢慢凸显出来

HTTP/2

Google开发了自己的浏览器Chrome，然后推出了新的SPDY协议。并用在自家的服务器上。相较1.1有以下改善：

  1. 二进制协议，不再是纯文本
  2. 可发起多个请求，废弃了1.1里的管道
  3. 使用专用算法压缩了头部，减少了数据传输量
  4. 允许服务器主动向客户端推送数据
  5. 增强了安全性，事实上要求加密通信

因为HTTP/1.1实现太经典，现在绝大部分请求仍旧是HTTP/1.1

HTTP/3

2都没普及，3就已经快出来了。Google革了自己的命，推出了QUIC，还是在自家的浏览器和服务器上玩。底层基于UDP向上层提供了稳定和可靠性的连接。
